# 프로젝트 시작 전 개발 환경 안내

## 개발 환경 개요

이 프로젝트는 **사내 폐쇄망** 환경을 타겟으로 하며, 개발은 사외 개인 PC에서 진행합니다.

---

## 인프라 구성

| 구분 | 내용 |
|------|------|
| 프론트엔드 런타임 | Nginx (Alpine) on Kubernetes |
| 백엔드 | Django REST API on Kubernetes |
| 데이터베이스 | Oracle DB (사내) |
| CI/CD | Jenkins (사내 GitHub 푸시 시 자동 빌드/배포) |
| 컨테이너 이미지 | node:22, nginx:alpine |

---

## 개발 흐름

```
[사외 개인 PC]
  Claude Code로 React 앱 개발
  → 사외 GitHub push

[사내 로컬 PC]
  사외 GitHub pull → 환경 맞게 수정
  → 사내 GitHub push

[사내 CI/CD]
  Jenkins 자동 빌드 → Docker 이미지 생성 → K8s 배포
```

---

## 기술 스택

- **Frontend**: React, TypeScript, Tailwind CSS, shadcn/ui, Vite
- **상태관리**: zustand
- **차트**: recharts
- **폰트**: Pretendard
- **빌드**: Vite (`npm run build`)

---

## 환경별 개발 방식

### 사외 (개발 단계)
- `npm run dev` 로 개발 및 테스트
- `npm run build` 로 빌드 확인
- Docker / Kubernetes 불필요
- 백엔드 API 접근 불가 → **MSW(Mock Service Worker)** 로 대체

### 사내 (배포 단계)
- Jenkins가 Docker 빌드 및 K8s 배포 자동 처리
- 실제 Django API 및 Oracle DB 연결

---

## Mock 환경 (MSW)

사외에서는 실제 API에 접근할 수 없으므로 MSW를 사용합니다.

- `src/mocks/handlers.ts` — API 핸들러 정의
- `src/mocks/browser.ts` — MSW 브라우저 설정
- 환경변수로 mock/real 전환:

```
VITE_USE_MOCK=true   # 사외 개발 시
VITE_USE_MOCK=false  # 사내 배포 시
```

API 스펙은 사내 Swagger 문서 기반으로 작성합니다.

| 엔드포인트 | 메서드 | 설명 | 주요 응답 필드 |
|-----------|--------|------|--------------|
| `{{/api/...}}` | `{{GET}}` | `{{설명}}` | `{{필드명}}` |

- 에러 응답 형식: `{{예: { code, message }}}`

| 변수명 | 사외 값 | 설명 |
|--------|--------|------|
| `VITE_USE_MOCK` | `true` | mock 활성화 여부 |
| `VITE_API_URL` | `{{사내 API URL}}` | Django API base URL |

## 사외 → 사내 이동 가이드

### 1단계 — 복사할 파일

사외 GitHub에서 clone 또는 zip 다운로드 후 아래 파일만 사내 PC로 가져옵니다.

```
✅ 복사
├── src/
├── public/
├── index.html
├── package.json
├── package-lock.json
├── vite.config.ts
├── tailwind.config.ts
├── postcss.config.js
├── components.json
├── tsconfig.json
├── tsconfig.app.json
├── tsconfig.node.json
└── eslint.config.js

❌ 제외
├── node_modules/   → 사내에서 npm install로 재설치
├── dist/           → 사내에서 새로 빌드
├── .git/           → 사내 git 새로 init
└── .github/        → Jenkins 사용하므로 불필요
```

> **폐쇄망으로 npm 접근 불가한 경우**
> `node_modules/` 도 함께 복사하면 `npm install` 없이 바로 빌드 가능

---

### 2단계 — 환경변수 생성

사내 PC에서 `.env.production` 파일을 직접 생성합니다. (`.gitignore`에 포함되어 있어 GitHub에 올라가지 않음)

```
VITE_USE_MOCK=false
VITE_API_URL=http://실제-사내-django-api-주소
```

---

### 3단계 — 빌드 확인

```bash
# node_modules 없이 복사한 경우
npm install

# 빌드
npm run build
```

`dist/` 폴더가 생성되면 성공.

---

### 4단계 — 사내 GitHub push

```bash
git init
git remote add origin http://사내-github/저장소.git
git add .
git commit -m "Initial"
git push -u origin main
```

이후 Jenkins가 자동으로 Docker 빌드 → K8s 배포를 처리합니다.
